# GMC-16 – настоящий машинный код

**GMC-16** — это 16-битный виртуальный процессор для обучения машинному коду.  
Вы пишете программы **цифрами** (опкоды, регистры, адреса), без ассемблера.

## Особенности

- 16‑бит, 64 КБ памяти  
- 8 регистров (R0–R7)  
- 5‑байтовые инструкции, опкод 5 бит  
- Абсолютная адресация (LOAD/STORE)  
- Порты ввода/вывода:  
  - `0xFFF0` – вывод символа  
  - `0xFFF1` – ввод символа (блокирующий)  
- Условные переходы: JEQ, JNE, JLT, JGE  
- Флаги: Z (ноль), C (перенос), E (равно), L (меньше)

## Команды (опкоды – десятичные)

| Опкод | Команда | Формат (5 чисел) | Действие |
|-------|---------|------------------|----------|
| 10 | MOV | `10 reg_dst reg_src imm16 0` | Rd = Rs или Rd = imm16 |
| 6  | ADD | `6 reg_dst reg_src 0 0 0` | Rd = Rd + Rs |
| 18 | SUB | `18 reg_dst reg_src 0 0 0` | Rd = Rd - Rs |
| 2  | AND | `2 reg_dst reg_src 0 0 0` | Rd = Rd & Rs |
| 26 | LOAD | `26 reg_dst 0 addr_lo addr_hi 0` | Rd = mem[addr] |
| 30 | STORE | `30 0 reg_src addr_lo addr_hi 0` | mem[addr] = Rs |
| 14 | CMP | `14 reg_dst reg_src 0 0 0` | сравнить Rd и Rs |
| 22 | JMP | `22 0 0 addr_lo addr_hi 0` | PC = addr |
| 21 | JEQ | `21 0 0 addr_lo addr_hi 0` | прыжок если равно |
| 23 | JNE | `23 0 0 addr_lo addr_hi 0` | прыжок если не равно |
| 24 | JLT | `24 0 0 addr_lo addr_hi 0` | прыжок если меньше |
| 25 | JGE | `25 0 0 addr_lo addr_hi 0` | прыжок если >= |
| 31 | HLT | `31 0 0 0 0 0` | останов |

## Запуск

### Без установки (требуется Python)
```bash
python run.py examples/hello.gmc
```

### Скомпилированный эмулятор
1. Запустите `build_exe.bat` (Windows) или `build_exe.sh` (Linux/Mac).  
2. Дважды кликните `launch.bat` (Windows) или `./launch.sh` (Linux/Mac).  
3. Выберите номер программы.

### Установщик Windows
- Запустите `GMC16_Setup.exe` (создаётся Inno Setup).  
- После установки появится ярлык "GMC-16 Launcher".

## Создание своих программ

1. Каждая строка – ровно 5 чисел (0–255).  
2. Комментарии начинаются с `;`.  
3. Числа можно писать в десятичном виде или шестнадцатеричном с префиксом `0x`.  
4. Программа загружается в память с адреса `0x4000`.  
5. Для переходов (JMP, JEQ и др.) нужно вычислять абсолютный адрес команды (например, если ваша метка находится на смещении 5 команд от начала, адрес = 0x4000 + 5*5).
