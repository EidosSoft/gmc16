## 1️⃣ Первое число в строке 

| Если ты видишь это число | То это значит | Команда | Что она делает |
|--------------------------|---------------|---------|----------------|
| **2** | AND | `AND Rd, Rs` | Побитовое И: Rd = Rd & Rs |
| **6** | ADD | `ADD Rd, Rs` | Сложение: Rd = Rd + Rs |
| **10** | MOV | `MOV Rd, Rs` или `MOV Rd, #imm` | Копирование из регистра или загрузка константы |
| **14** | CMP | `CMP Rd, Rs` | Сравнить Rd и Rs (установить флаги E и L) |
| **18** | SUB | `SUB Rd, Rs` | Вычитание: Rd = Rd - Rs |
| **21** | JEQ | `JEQ address` | Прыжок, если флаг E = 1 (равно) |
| **22** | JMP | `JMP address` | Безусловный прыжок |
| **23** | JNE | `JNE address` | Прыжок, если флаг E = 0 (не равно) |
| **24** | JLT | `JLT address` | Прыжок, если флаг L = 1 (меньше) |
| **25** | JGE | `JGE address` | Прыжок, если флаг L = 0 (больше или равно) |
| **26** | LOAD | `LOAD Rd, [address]` | Загрузить 16-битное число из памяти (или ввод с клавиатуры) |
| **30** | STORE | `STORE Rs, [address]` | Записать 16-битное число в память (или вывод на экран) |
| **31** | HLT | `HLT` | Остановить программу |
| Любое другое число от 0 до 255 (кроме перечисленных) | Ошибка! Эмулятор не знает такой команды | — | — |

## 2️⃣ Второе число в строке (упакованные регистры `reg_dst` и `reg_src`)

Второе число вычисляется так: `reg_dst * 16 + reg_src`.  
**reg_dst** – куда класть результат (0..7)  
**reg_src** – откуда брать (0..7)

Ниже таблица для **всех возможных комбинаций**, которые ты можешь использовать.  
Если ты видишь число – смотри, какие регистры оно означает.

| 0 | 0 | 0 | R0 <- R0 (бесполезно) или загрузка константы в R0 | `MOV R0, #imm` или `MOV R0, R0` |

| 2 | 0 | 2 | R0 <- R2 | `MOV R0, R2` |

## 3️⃣ Третье и четвёртое числа (младший и старший байт 16-битного числа)

Эти числа вместе образуют **одно число** по формуле: `третье_число + четвертое_число * 256`.  
Ниже таблица **самых частых значений** (что они значат в десятичной, шестнадцатеричной и как символ или адрес).

| Десятичное | Шестнадцатеричное | Что это значит | Когда используется |
|------------|-------------------|----------------|---------------------|
| 0 | 0x00 | ноль | очистка регистра, нулевой адрес |
| 1 | 0x01 | единица | инкремент (если добавить в R3=1) |
| 2 | 0x02 | двойка | ... |
| 10 | 0x0A | десять | маленькие константы |
| 48 | 0x30 | символ '0' | вывод цифр |
| 49 | 0x31 | символ '1' | ... |
| 65 | 0x41 | символ 'A' | вывод букв |
| 66 | 0x42 | 'B' | |
| 90 | 0x5A | 'Z' | |
| 97 | 0x61 | 'a' | |
| 122 | 0x7A | 'z' | |
| 240 | 0xF0 | младший байт порта вывода | вместе со старшим 255 даёт адрес 0xFFF0 |
| 241 | 0xF1 | младший байт порта ввода | вместе с 255 даёт 0xFFF1 |
| 255 | 0xFF | старший байт для портов | 255*256 = 65280, +240 = 65520 (0xFFF0) |

**Примеры:**  
- Если третье = `65`, четвертое = `0` → число = 65 (символ 'A')  
- Если третье = `240`, четвертое = `255` → число = 240 + 255*256 = 65520 (адрес 0xFFF0)  
- Если третье = `0`, четвертое = `64` → число = 0 + 64*256 = 16384 (адрес 0x4000)

## 4️⃣ Пятое число в строке (резерв)

Всегда должно быть **0**.  
Если ты видишь другое число – эмулятор его проигнорирует, но лучше так не делать.

| Число | Что значит |
|-------|-------------|
| 0 | Всё правильно |
| 1..255 | Ошибка в написании программы (неправильно) |

---

## 🧪 Примеры: разбор готовых строк

### Строка `10 16 240 255 0`

| Позиция | Число | Что значит |
|---------|-------|-------------|
| 1 | 10 | команда MOV |
| 2 | 16 | reg_dst=1, reg_src=0 → MOV R1, #... |
| 3 | 240 | младший байт = 240 |
| 4 | 255 | старший байт = 255 |
| 5 | 0 | резерв |
| **Итог** | | **MOV R1, #0xFFF0** |

### Строка `30 0 0 240 255 0`

| Позиция | Число | Что значит |
|---------|-------|-------------|
| 1 | 30 | STORE |
| 2 | 0 | reg_dst=0, reg_src=0 → сохраняем из R0 |
| 3 | 240 | младший байт адреса |
| 4 | 255 | старший байт адреса = 0xFFF0 |
| 5 | 0 | резерв |
| **Итог** | | **STORE R0, [0xFFF0]** (вывод символа из R0) |

### Строка `6 0 1 0 0 0`

| Позиция | Число | Что значит |
|---------|-------|-------------|
| 1 | 6 | ADD |
| 2 | 0 | reg_dst=0, reg_src=1 → R0 = R0 + R1 |
| 3 | 0 | не используется |
| 4 | 0 | не используется |
| 5 | 0 | резерв |
| **Итог** | | **ADD R0, R1** |

### Строка `31 0 0 0 0 0`

| Позиция | Число | Что значит |
|---------|-------|-------------|
| 1 | 31 | HLT (стоп) |
| 2-5 | 0 | игнорируются |
| **Итог** | | **HLT** |

## 📌 Самое главное (запомни)

- **Первое число** – что делать (смотри таблицу опкодов)
- **Второе число** – откуда и куда (регистры, вычисляется как `reg_dst*16+reg_src`)
- **Третье + четвёртое** – число или адрес (младший байт, потом старший)
- **Пятое** – всегда 0
